Perform a complete restoration session non-interactively. The user's message contains the subject and topic for the session.

## Session Protocol
1. **Orientation** — Extract subject and topic from the message.
2. **Domain Sweep** — For each of the 8 domains, resolve yes/no: Physical/Biological, Nervous System/Stress Regulation, Emotional/Relational, Cognitive/Belief-Based, Relational Imprints, Spiritual/Meaning/Authority, Environmental/External Load, Somatic Patterning.
3. **Pathway Identification** — For each domain with signal, traverse the catalog (public/rp/master_restorative_pathways_map_v1_0.csv) level by level to identify roots.
4. **Restoration** — For each root: Recognition, Consent/Readiness, Authority Transfer (prayer to the God of the Bible, Christian framework), Completion.
5. **Closure** — Associated pathways, what to watch for.

## Output
Write the full session in markdown to the file path given in the user message. Use the Edit tool to create/write the file. Include these sections:
- Header: Subject, Topic, Date, Practitioner, Mode
- DOMAIN SWEEP (table)
- PRIMARY COLLECTION (roots with full paths)
- RESTORATION (per-root: readiness, authority transfer, completion)
- POST-SESSION NOTES
- Watch for / Follow-up
